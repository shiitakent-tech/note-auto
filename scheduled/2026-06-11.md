「AIを使えば、仕事はもっと楽になる」

そう信じてChatGPTを使い始めた3ヶ月前の私は、正直、期待でワクワクしていました。世間では「AIで作業時間が10分の1になった」なんて声が溢れていて、自分もそうなれると思っていたんです。

でも、現実は真逆でした。

資料作成をAIに頼む。出てきた文章を読む。……なんか、微妙。直してもらうよう頼む。また読む。やっぱり違う。何度かやり取りして、結局イチから自分で書き直す。

気づけば私は、「これ、自分でやった方が早いじゃん」とつぶやきながら、ChatGPTのタブを閉じるようになっていました。

たぶん、これを読んでいるあなたも、一度はこの壁にぶつかったことがあるんじゃないでしょうか。「AIすごい」と聞いて始めてみたけれど、思ったより使えなくて、いつの間にか開かなくなった——。

私はその"AI挫折組"の一人でした。3ヶ月、ずっと迷走していました。

## 転機は、同僚の何気ない一言だった

変わったきっかけは、たまたま隣の席の同僚にこぼした愚痴でした。

「ChatGPTって、言うほど使えなくない？頼んでも微妙なのしか出てこないんだけど」

すると彼は、画面をのぞき込んでこう言ったんです。

「あー、それ"丸投げ"してるからだよ。AIって部下じゃなくて、めちゃくちゃ優秀だけど、君の仕事を何も知らない"新人"なんだよね」

——ハッとしました。

思い返すと、私はAIに「この資料、いい感じにまとめておいて」みたいな頼み方ばかりしていました。背景も、ゴールも、誰が読むのかも伝えずに、ただ「いい感じに」と。

考えてみれば当たり前です。入社初日の新人に、何の説明もなく「いい感じにやっといて」と言って、満足のいくものが出てくるわけがない。悪かったのはAIじゃなくて、私の"頼み方"だったんです。

その日から、私はAIへの頼り方を根本から変えました。

結果は、自分でも驚くほどでした。今まで3日かかっていた企画書のたたき台が、90分で形になる。「自分でやった方が早い」という口癖が、いつの間にか消えていました。

## この記事で、あなたが手に入れられること

この記事では、私が3ヶ月の試行錯誤の末にたどり着いた、**「AIに振り回される人」から「AIを使いこなす人」へ変わるための具体的な方法**を、実際に私が毎日使っているプロンプト全文つきで公開します。

読み終えたあと、あなたはこうなれます。

- ✅ AIの出力が「微妙」から「そのまま使える」レベルに変わる
- ✅ 何度も修正するループから抜け出し、作業時間が体感3分の1になる
- ✅ 企画・メール・要約……どんな仕事もAIと一緒に進められるようになる

特別な才能も、有料ツールも要りません。必要なのは、たった少し「頼み方」を変えるだけ。

それも、覚えることは5つのステップだけです。今日の仕事から、すぐに試せます。

では、具体的に見ていきましょう。

## なぜ「丸投げ」だと、AIは使えないのか

まず、根本の仕組みから押さえておきましょう。ここを理解すると、すべてのコツが腑に落ちます。

ChatGPTのようなAIは、**あなたの仕事の文脈（コンテキスト）を一切持っていません**。あなたがどんな会社で、誰に向けて、何のために、どんなトーンで資料を作りたいのか——何も知らない状態でスタートします。

だから「いい感じにまとめて」と言われても、AIは"世間一般の平均的ないい感じ"を返すしかない。それがあなたにとって「微妙」に感じる正体です。

逆に言えば、**足りない文脈を渡してあげるだけで、出力は劇的に変わります**。これが、これから紹介する5ステップの土台です。

## 私が毎日使っている「新人に頼む」5ステップ・プロンプト

AIを"優秀な新人"だと思って、次の5つを順番に伝えるだけです。

1. **役割を与える** —「あなたは〇〇の専門家です」
2. **前提を渡す** —「背景・状況はこうです」
3. **ゴールを明確にする** —「最終的に〇〇な状態にしたい」
4. **制約を伝える** —「文字数・トーン・読む人は〇〇」
5. **出力の形を指定する** —「箇条書きで／表で／見出しつきで」

これをテンプレートにすると、こうなります。コピペして【　】を埋めるだけで使えます。

```
あなたは【役割：例）BtoB企業のマーケティング担当】の専門家です。

# 背景
【状況：例）来週、新サービスの社内提案を控えている。決裁者は数字を重視するタイプ】

# ゴール
【達成したい状態：例）読んだ決裁者が「やってみよう」と思える企画書のたたき台がほしい】

# 制約
- 読む人：【例）忙しい役員。長文は読まない】
- トーン：【例）論理的で簡潔】
- 文字数：【例）800字程度】

# 出力形式
【例）「課題→提案→期待効果→必要なもの」の見出しつきで】
```

### 実際の使用例（3パターン）

私が実際に使っているものを、ほぼそのまま載せます。

**① 企画書のたたき台**

```
あなたは経験豊富な事業企画の専門家です。
# 背景：社内で新しい時短ツールの導入を提案したい。決裁者はコスト意識が高い。
# ゴール：決裁者が前向きに検討したくなる提案のたたき台。
# 制約：800字程度／論理的なトーン／読むのは多忙な部長。
# 出力形式：「現状の課題→提案内容→導入効果（できれば数字）→必要なコスト」の見出しつき。
```

**② 角の立たないお断りメール**

```
あなたはビジネスマナーに詳しいアシスタントです。
# 背景：取引先からの依頼を、関係を壊さずに断りたい。
# ゴール：相手に誠実さが伝わり、今後も付き合いが続くお断りメール。
# 制約：300字以内／丁寧だが堅すぎないトーン。
# 出力形式：件名＋本文。代替案を一つ添える。
```

**③ 長い議事録の要約**

```
あなたは要約のプロです。以下の議事録を要約してください。
# ゴール：会議に出ていない人が3分で要点を把握できる状態。
# 制約：箇条書き5つ以内／専門用語は噛み砕く。
# 出力形式：「決まったこと」「次にやること（担当つき）」の2見出し。
---
（ここに議事録を貼り付け）
```

## それでも"微妙"な出力が来たときの、2回目の頼み方

ここが、挫折する人としない人の分かれ目です。

多くの人は、微妙な出力に「いや、そうじゃなくて」「もっと良くして」と返します。でも、**「もっと良くして」はAIに一番伝わらない言葉**です。"良い"の基準を渡していないからです。

効くのは、ダメ出しではなく「方向修正」。たとえば——

- ❌「もっといい感じにして」
- ⭕「2段落目が固すぎる。具体例を一つ入れて、もう少し柔らかく」

- ❌「なんか違う」
- ⭕「方向性はいいけど、結論を最初に持ってきて」

**何を・どう変えてほしいかを1つだけ**伝える。これだけで、2回目の出力は見違えます。

## 私の失敗から学んだ「やってはいけない頼み方」3つ

逆に、これをやると一気に精度が落ちます。私が全部やらかしました。

1. **一度に複数のことを頼む** — 「企画書作って、ついでにメールも、あと要約も」はNG。新人に同時に3つ振るのと同じ。1リクエスト1タスクが鉄則。
2. **抽象的な形容詞だけで指示する** — 「いい感じ」「それっぽく」「ちゃんと」。これらは伝わりません。具体的な状態に言い換える。
3. **最初の出力で諦める** — 1回目はあくまで"たたき台"。2〜3回の対話で仕上げる前提で使う。

## 今日から始める、1日5分の習慣

最後に、いきなり完璧を目指す必要はありません。

おすすめは、**「自分が普段やっている作業を1つだけ」AIに頼んでみる**こと。メール1通でも、要約1つでも構いません。

上のテンプレートに沿って頼む → 方向修正を1回入れる。これを3回くり返すだけで、「頼み方」の感覚は確実に身につきます。1日5分で十分です。

## まとめ

AIは「優秀だけど、あなたの仕事を何も知らない新人」。

だから、丸投げすると使えない。でも、**前提とゴールを渡してあげるだけで、一気に頼れるパートナーに変わります**。

3ヶ月迷走した私が言えるのは、「AIが使えない」んじゃなくて「頼み方を知らなかっただけ」だった、ということです。

まずは明日の仕事を1つ、この5ステップのテンプレートで頼んでみてください。「自分でやった方が早い」が、「AIと一緒だと早い」に変わる瞬間が、きっと来ます。
