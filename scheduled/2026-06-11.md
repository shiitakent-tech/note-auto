ちょっと、私の3ヶ月前の話を聞いてくれますか。

「AIを使えば、仕事はもっと楽になるよ」——まわりがそう言うのを真に受けて、私、けっこう本気でワクワクしてたんです。SNSを開けば「AIで作業時間が10分の1になった」なんて景気のいい話ばかり。よし、自分もそうなるぞ、って。

……で、どうなったと思います？

正直に言いますね。**逆に、忙しくなりました。**

資料づくりをChatGPTに頼む。出てきた文章を読む。うーん、なんか、微妙。「もうちょっといい感じに」って直してもらう。また読む。やっぱりしっくりこない。何往復かして、結局ため息ついて、自分で一から書き直す——。

そんなことを繰り返してるうちに、私、いつの間にかこう独り言を言うようになってました。

「……これ、自分でやったほうが早いやん」

たぶん、これ読んでくれてるあなたも、ちょっと身に覚えがあるんじゃないですか？「AIすごいらしい」って聞いて始めてみたけど、思ったより使えなくて、気づけば開かなくなってた。あの感じ。

そう、私はみごとに"AI挫折組"の一人だったんです。3ヶ月、ずーっと空回りしてました。

## 救ってくれたのは、隣の席のひと言だった

転機って、案外しょうもないことからやってくるんですよね。

ある日、隣の席の同僚に、ぽろっと愚痴をこぼしたんです。「ChatGPTって、言うほど使えなくない？頼んでも微妙なのしか返ってこないんだけど」って。

そしたら彼、私の画面をのぞき込んで、こう言ったんです。

「あー……それ、"丸投げ"してるからだよ。AIってさ、部下じゃないんだよね。めちゃくちゃ優秀なんだけど、君の仕事を何ひとつ知らない"新人"なんだよ」

——その瞬間、頭の中で何かがカチッとはまった気がしました。

だって、思い返してみたら、私の頼み方ってずっとこんな感じだったんです。「この資料、いい感じにまとめといて」。背景も、ゴールも、誰が読むのかも、なんにも言わずに、ただ「いい感じに」。

……いやこれ、よく考えたら無茶ですよね。入社初日の新人さんに、何の説明もなく「いい感じにやっといて」って言って、ちゃんとしたものが出てくるわけがない。

悪かったのはAIじゃなくて、私の"頼み方"だったんです。気づいた時、ちょっと恥ずかしくなりました。

で、その日から、頼り方をまるっと変えてみたんですね。そしたら——自分でもびっくりするくらい変わりました。3日うんうん唸ってた企画書のたたき台が、90分で形になる。あの口癖、「自分でやったほうが早い」が、いつの間にかどこかへ消えていました。

## この記事で、あなたに持って帰ってほしいもの

ここから先で、私がこの3ヶ月の遠回りの末にたどり着いた、**「AIに振り回される側」から「AIを使いこなす側」に変わるためのやり方**を、ぜんぶお話しします。きれいごとじゃなくて、私が今も毎日使ってるプロンプト、そのまま載せます。

読み終わったころには、あなたはきっとこうなってます。

- ✅ AIの返事が「微妙……」から「お、そのまま使えるじゃん」に変わる
- ✅ 何度も直すあの不毛なループから、すっと抜け出せる
- ✅ 企画でもメールでも要約でも、「とりあえずAIに相談しよ」が口グセになる

特別な才能も、高い有料ツールも、いりません。ほんの少し「頼み方」を変えるだけ。覚えることも、たった5つです。今日の仕事から、すぐ試せますよ。

肩の力を抜いて、いきましょう。

## まず、なんで「丸投げ」だとダメなのか

コツの話に入る前に、ひとつだけ仕組みを押さえさせてください。ここが腑に落ちると、あとの話が全部すっと入ってきます。

ChatGPTみたいなAIって、**あなたの仕事の事情を、これっぽっちも知らない**んです。あなたがどんな会社で働いてて、誰に向けて、何のために、どんな空気感の資料を作りたいのか——スタート時点では、ほんとに何も知らない。まっさらです。

だから「いい感じにまとめて」と言われても、AIには"世間でいうところの、平均的ないい感じ"しか返せない。それがあなたに「なんか違う」と感じさせてる正体なんですね。

逆に言うと——**足りてない事情を、ちょっと渡してあげるだけ**。それだけで、返ってくるものが別物に変わります。これが、これから話す5ステップの土台です。

## 私が毎日やってる「新人に頼む」5ステップ

AIを"優秀だけど何も知らない新人"だと思って、次の5つを順番に伝える。ほんとに、これだけです。

1. **役割をあげる** —「あなたは〇〇の専門家です」
2. **事情を渡す** —「いまこういう状況なんです」
3. **ゴールを見せる** —「最終的にこうなってると嬉しい」
4. **しばりを伝える** —「文字数・トーン・読む人はこんな感じ」
5. **形を指定する** —「箇条書きで／表で／見出しつきで」

テンプレートにすると、こんな感じ。【　】を埋めて、コピペするだけで使えます。

```
あなたは【役割：例）BtoB企業のマーケ担当】の専門家です。

# 背景
【状況：例）来週、新サービスの社内提案がある。決裁者は数字を見るタイプ】

# ゴール
【こうなりたい：例）読んだ決裁者が「やってみるか」と思える企画のたたき台がほしい】

# しばり
- 読む人：【例）忙しい役員。長い文章は読まない】
- トーン：【例）論理的だけど、固すぎない】
- 文字数：【例）800字くらい】

# 出力の形
【例）「課題→提案→期待できる効果→必要なもの」の見出しつきで】
```

### 実際に私が使ってるやつ（3つ）

きれいに整えたものじゃなくて、ほぼ普段のまま載せますね。

**① 企画書のたたき台がほしいとき**

```
あなたは経験豊富な事業企画の専門家です。
# 背景：社内で新しい時短ツールの導入を提案したい。決裁者はコスト意識が強め。
# ゴール：決裁者が前向きに検討したくなる、提案のたたき台。
# しばり：800字くらい／論理的なトーン／読むのは多忙な部長。
# 出力の形：「いまの課題→提案内容→導入効果（できれば数字）→かかるコスト」の見出しつき。
```

**② 角を立てずにお断りしたいとき**

```
あなたはビジネスマナーに詳しいアシスタントです。
# 背景：取引先からの依頼を、関係を壊さずにやんわり断りたい。
# ゴール：誠実さが伝わって、これからも付き合いが続くお断りメール。
# しばり：300字以内／丁寧だけど堅すぎないトーン。
# 出力の形：件名＋本文。代わりの案をひとつ添えて。
```

**③ 長い議事録を要約したいとき**

```
あなたは要約のプロです。次の議事録を要約してください。
# ゴール：会議に出てない人が、3分で要点をつかめる状態。
# しばり：箇条書き5つ以内／専門用語はかみ砕く。
# 出力の形：「決まったこと」「次にやること（担当つき）」の2つの見出しで。
---
（ここに議事録を貼る）
```

## それでも"微妙"なのが返ってきたら——2回目の頼み方

正直、ここがいちばん大事です。挫折する人としない人の分かれ道、たぶんここ。

多くの人は、微妙な返事に「いや、そうじゃなくて」「もっと良くして」って返しちゃうんですよね。私もそうでした。でも——**「もっと良くして」って、AIに一番伝わらない言葉**なんです。だって、"良い"の基準を渡してないから。

効くのは、ダメ出しじゃなくて「方向を直す」言い方。たとえば、こんなふうに。

- ❌「もっといい感じにして」
- ⭕「2段落目がちょっと固いかな。具体例をひとつ入れて、もう少しやわらかく」

- ❌「なんか違うんだよな」
- ⭕「方向性はいい感じ。ただ、結論を最初に持ってきてくれる？」

**何を、どう変えてほしいかを、ひとつだけ**。これだけで、2回目はほんとに見違えますよ。

## 私がやらかした「これはダメ」な頼み方3つ

逆に、これやると一気に精度が落ちます。……ええ、全部、私が通った道です。

1. **一度にあれもこれも頼む** —「企画書作って、ついでにメールも、あと要約も」。これNG。新人さんに同時に3つ振るのと同じです。1回につき1つ。
2. **ふわっとした言葉だけで指示する** —「いい感じ」「それっぽく」「ちゃんと」。伝わりません。具体的な状態に言い換えてあげて。
3. **1回目で諦める** — 最初の返事は、あくまで"たたき台"。2〜3回やりとりして仕上げるもの、くらいに思っておくと気が楽です。

## 今日からの、1日5分でいい習慣

最後に。いきなり完璧を目指さなくて大丈夫です。むしろ、目指さないでください。

おすすめは、**「いつもやってる作業を、ひとつだけ」AIに頼んでみる**こと。メール1通でも、要約ひとつでも、なんでもいいんです。

さっきのテンプレートに沿って頼んで、方向修正を1回入れる。これを3回くりかえすだけ。それだけで、「頼み方」の感覚って、ほんとに体に入ってきます。1日5分で十分。

## おわりに

AIは、「優秀だけど、あなたの仕事を何も知らない新人」。

だから、丸投げすると使えない。でも——**事情とゴールを渡してあげるだけで、びっくりするくらい頼れる相棒に変わります。**

3ヶ月さんざん遠回りした私が言えるのは、これだけです。「AIが使えない」んじゃなかった。私が「頼み方を知らなかった」だけだったんです。

だまされたと思って、まずは明日の仕事をひとつ、この5ステップで頼んでみてください。「自分でやったほうが早い」が、「AIと一緒だと、早い」に変わる瞬間。きっと、来ますから。
