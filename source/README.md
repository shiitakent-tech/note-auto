# NotebookLM → Claude → note ワークフロー

本や長い資料を **NotebookLM** で読み込み、要点＋出典を抽出 → このフォルダに保存 →
`from_source.py` で **Claude** が note 記事ドラフトに仕上げます。

NotebookLM と Claude を直接つなぐ公式連携は無いので、橋渡しは「コピペ」です。
このフォルダがその受け渡し場所になります。

---

## 手順

### 1. NotebookLM 側：要点＋出典を抽出する

NotebookLM に本/資料（PDF・URL・テキスト）をソースとして追加し、チャットに
**下のプロンプトをそのまま貼り付け**ます。

```
このソースから、note記事の素材として使える「要点」を抽出してください。
次の形式で、Markdownで出力してください（あなたの感想は不要、ソースに基づく事実のみ）:

# 資料タイトル / 著者
# この資料の一言要約（1〜2文）

# 主要な要点（5〜8個）
各要点について:
- 要点（1〜2文で言い切る）
- 出典（章・節・ページ・該当箇所）
- 補足（具体例・数字・引用があれば短く）

# 読者がすぐ試せる実践ポイント（3個以上）
- 行動 / その根拠（出典）

# 印象的な一節（短い引用を2〜3個、出典つき）
```

> 出典を必ず付けさせるのがコツです。Claude 側はこの出典を記事に織り込み、
> 「メモに無いことは書かない」根拠として使います。

### 2. 抽出結果をこのフォルダに保存

NotebookLM の出力をコピーして、`source/` に `.md` で保存します。
例: `source/ai_work_book.md`（テンプレは `_template.md`）

### 3. Claude で記事ドラフト化

```bash
python from_source.py source/ai_work_book.md          # 有料(デフォルト300円)
python from_source.py source/ai_work_book.md --price 0 # 無料記事
```

→ `drafts/from_source_xxx_YYYYMMDD_HHMM.md` にレビュー用ドラフトが出来ます。
**自動投稿はされません。**

### 4. レビューして予約投稿に回す

中身を確認し、問題なければ予約フォルダに移すだけ:

```bash
cp drafts/from_source_xxx_YYYYMMDD_HHMM.md scheduled/2026-06-20.md
```

→ 既存パイプライン（GitHub Actions、朝8時/夜19時 JST）が、その日付の記事を自動投稿します。

---

## メモ
- 出典の薄い素材だと記事も浅くなります。NotebookLM 側で要点を厚めに出すほど良い記事になります。
- 価格や有料ラインの位置は、生成後にドラフトを直接編集して調整できます。
